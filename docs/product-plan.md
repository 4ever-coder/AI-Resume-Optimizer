# 产品方案

## 产品定位

AI Resume Optimizer 简历优化器服务于第一次做简历优化、不了解招聘筛选逻辑的小白用户。系统把“上传简历 -> 填写目标岗位 -> 自助连接模型 -> 生成报告 -> 复制或导出建议”做成一个完整流程，降低用户理解和使用门槛。

## 核心用户流程

1. 上传 PDF、DOCX 或 TXT 简历。
2. 填写岗位名称，必要时粘贴目标 JD。
3. 在模型设置中填写自己的 API Key、Base URL 和模型名。
4. 点击开始分析。
5. 查看报告：总分、星级、多维评分、整体印象、风险提示和优化建议。
6. 复制建议、重新生成、下载报告，或回到历史记录查看旧报告。

## API Key 接入方案

- 用户 API Key 由前端表单输入，只随本次 `/api/analyses` 请求传给后端。
- 后端用该 Key 调用 OpenAI-compatible Chat Completions，不写入日志、不持久化、不返回给前端。
- Base URL 默认使用 OpenAI 官方兼容地址，用户可改成其他兼容服务。
- 前端明确提示用户使用自己的 Key，并在分析完成后允许一键清空本地设置。
- MVP 不做平台统一 Key，避免产生不可控成本和密钥托管风险。

## 模型推荐方案

- 默认推荐 `gpt-4.1-mini`：适合高性价比结构化分析。
- 高质量模式推荐 `gpt-4.1`：适合重要岗位和复杂经历。
- 兼容模式允许用户填写任意 OpenAI-compatible 模型名。

## 报告结构

报告稳定输出以下字段，前端按字段直接渲染：

- `summaryScore`：0-100 总分。
- `starLevel`：1-5 星级。
- `position`：岗位名称或岗位方向。
- `overallComment`：整体评价。
- `dimensions`：岗位匹配度、能力深度、表达质量、风险提示等维度评分。
- `impressions`：整体印象标签和说明。
- `suggestions`：精简、扩展、补充、重写、AI 融合等建议卡片。
- `rewriteSamples`：可直接复制的改写样例。
- `riskWarnings`：真实性、空窗、数据缺失、岗位不匹配等风险提示。

## 任务边界

MVP 先保证个人用户可以完整完成一次简历分析和报告导出。账号系统、团队管理、复杂计费、在线模板编辑和企业 ATS 对接留到后续版本。
