# Linear 子任务执行顺序

本项目按 Linear 子任务串行推进，避免多个功能并行修改导致验收边界不清。

1. `QIA-122`：产品方案与任务入口。
2. `QIA-123`：评分维度与报告 JSON 协议。
3. `QIA-124`：简历上传与文本解析。
4. `QIA-125`：历史记录页面。
5. `QIA-126`：模型设置页面。
6. `QIA-127`：Analysis Rules。
7. `QIA-128`：后端接口。
8. `QIA-129`：报告页面。
9. `QIA-130`：优化建议卡片。
10. `QIA-131`：导出与分享。
11. `QIA-132`：安全与成本控制。
12. `QIA-133`：测试与部署验收。

每个子任务完成后：

- 运行与该任务相关的验证命令。
- 展示 `git diff --stat`。
- 使用中文提交消息提交。
- 推送到 GitHub 远端。
- 写一份中文计划执行报告，说明功能变化和使用办法。
